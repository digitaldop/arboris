Conti business e corporate (TrueLayer)

TrueLayer espone retail, business e corporate come segmenti separati. Per includere anche i conti business delle banche non serve cambiare nulla qui: Arboris omette il parametro providers quando non c'e' un'istituzione specifica o una lista impostata nel campo qui sotto, lasciando che sia la tua app TrueLayer a decidere quali provider mostrare.

Come attivare i provider business/corporate sulla tua app:

  1. Apri la Console TrueLayer → la tua app (Arboris Banking).
  2. Vai in Data API → Providers (o Allowed providers).
  3. Abilita le banche desiderate per paese e per segmento (retail, business, corporate).
  4. Salva la configurazione.

In alternativa, se vuoi forzare via Arboris una lista precisa di providers (utile se piu' app condividono la stessa console), compila il campo "Providers TrueLayer" qui sotto con i provider_id desiderati separati da spazio, ad esempio: uk-ob-all uk-oauth-all it-ob-all fr-ob-all.

Enable Banking (PSD2 / AIS)

Crea l'applicazione nel Control Panel, carica la public key (certificato) e salva l'Application ID (UUID): va nel campo Secret ID / Client ID / App-id qui sotto.

Genera la coppia di chiavi RSA, conserva la private key PEM nel campo dedicato: Arboris la usa per firmare i JWT (qualsiasi chiamata all'API) con header kid = Application ID.

Il Secret Key / Client Secret in questa form non e' usato: lascialo vuoto. Registra il redirect URI (blocco arancione) nella console Enable Banking esattamente come mostrato; dopo il consenso l'utente torna su Arboris e il code della query viene scambiato con POST /sessions.

{% endif %} {% if is_saltedge_adapter %}

Configurazione Salt Edge (Account Information v6)

Salt Edge usa un Widget ospitato presso di loro: al momento della connessione Arboris chiama POST /connections/connect, riceve un connect_url e reindirizza l'utente lì. Alla fine dell'autenticazione forte (SCA) l'utente rientra sul nostro callback interno: nessun redirect URI da whitelistare nel pannello Salt Edge.

Credenziali richieste: App-id va nel campo Secret ID / Client ID / App-id, Secret va nel campo Secret Key / Client Secret / Secret. Le trovi nella sezione Secrets del tuo account Salt Edge.

Ambiente di test: attiva "Includi provider di test" qui sotto per vedere le fake banks di Salt Edge nel widget. Anche in stato Pending/Test, alcuni endpoint (tipicamente POST /connections/connect) possono rispondere con un 403 Forbidden lato WAF di Salt Edge se la richiesta non e' firmata. In quel caso genera una coppia di chiavi RSA, carica la public key nel dashboard Salt Edge ("Keys") e incolla la private key PEM nel campo qui sotto: Arboris firmera' automaticamente tutte le chiamate.

Come generare la coppia di chiavi RSA (esempio): 

openssl genrsa -out saltedge_private.pem 2048
openssl rsa -in saltedge_private.pem -pubout -out saltedge_public.pem

Carica saltedge_public.pem nel dashboard Salt Edge (sezione Keys / Public key), poi incolla il contenuto di saltedge_private.pem nel campo "Private key RSA PEM (Salt Edge)" qui sotto.
